# 阿岳 Sora2 角色上传工具 v4.x 使用说明

## 功能概览

支持上传 mp4 视频 或 图片（jpg/png）提取 Sora 角色 Character ID
图片会自动转为 4 秒静态视频 + 空白音轨
支持保存多个角色记录、本地查看 & 复制 @ID
支持与 Gitee 仓库双向同步（合并模式，避免覆盖）
可自定义 API Host、轮询间隔、最大轮询次数

## 关键设置（在界面最上方）

API Key：必填，从你的 Sora 服务提供方获取

API Host：默认 https://grsai.dakka.com.cn，可改

轮询间隔（秒）：建议 3~10 秒，太短可能被限流，太长等待时间长

最大轮询次数：建议 30~120，根据实际任务耗时调整

（保存配置后立即生效，下次启动也会记住）

时间范围说明

视频：填写如 0,3 表示从第0秒到第3秒提取（最多3秒）
图片：自动忽略输入，固定转为 4 秒视频

常见失败原因

"Upload may depict a realistic human..." → 画面含真人/写实人物 → 建议裁剪或换素材
超时 → 调大“最大轮询次数”或检查网络/API 是否正常
code != 0 → 检查 API Key 是否有效

云端同步操作

「下载云端角色（合并）」：拉取 Gitee → 与本地合并 → 保存
「推送本地到云端」：先备份本地 → 拉取最新云端 → 合并 → 推送到 Gitee

建议

首次使用请先设置 API Key 并保存
轮询参数不要设得过于激进（避免被 API 封）
cache 文件夹会保存缩略图和 sora_characters.json，不要随意删除
